# 2026-03-05 태깅 시스템 전면 개편 및 통합 고도화 상세 보고서

## 1. 개요
기존의 하드코딩된 Enum(`NoticeKeyword`) 기반 태깅 시스템을 데이터베이스 메타데이터 기반 아키텍처로 완전히 전환하고, 이를 바탕으로 공고 검색, 정렬, 캘린더 연동 및 전반적인 사용자 인터페이스(UI/UX)를 대대적으로 개선함.

---

## 2. 백엔드 아키텍처 혁신 및 고도화

### A. 메타데이터 기반 태깅 시스템 (Overhaul)
*   **Enum 탈피 및 DB화**: 유지보수가 어렵고 수천 줄에 달하던 `NoticeKeyword.java` Enum을 삭제하고, `notice_keyword_metadata` 테이블을 신설함.
*   **대규모 데이터 구축**: 전국의 시/도(METRO), 시/군/구(CITY_COUNTY), 지하철 호선(SUBWAY_LINE), 지하철 역명(STATION) 및 시스템 상태, 주거 유형 등 **총 1,244개**의 핵심 키워드를 DB화함.
*   **캐싱 전략**: 성능 최적화를 위해 `TagService` 기동 시 `@PostConstruct`를 활용하여 모든 메타데이터를 메모리(`List<NoticeKeywordMetadata>`)에 로드, 실시간 공고 분석 시 DB 부하를 제로화함.

### B. 태깅 엔진(`TagService`) 정밀화
*   **정규표현식(Regex) 제거**: `#근린생활시` 등 오탐을 유발하던 "한글+시/군/구" 패턴 매칭 로직을 완전히 제거하고, 100% 확정된 메타데이터 매칭 방식으로 전환하여 신뢰도 확보.
*   **이름 기반 중복 방지**: 동일한 이름의 태그(예: "청주")가 여러 타입(`REGION`, `CITY_COUNTY`)으로 중복 생성되는 현상을 방지하기 위해, 태그 추가 시 이름 중복 체크 로직을 강화함.
*   **시스템 상태 자동화**: `접수중`, `마감`, `결과발표`, `안내` 4개 상태를 `SYSTEM` 타입 태그로 관리. 수집 및 초기화 시점에 오늘 날짜와 마감일을 비교하여 자동으로 상태 태그를 부여/갱신함.

### C. 검색 및 정렬 로직 고도화 (QueryDSL)
*   **인기순 정렬 구현**: "관심공고순" 선택 시, 전체 사용자의 찜(하트) 개수를 서브쿼리로 계산하여 정렬하는 기능을 구현.
*   **필터 동기화**: 키워드 검색과 서클 필터(기관, 상태)가 유기적으로 결합되도록 `BooleanBuilder` 로직을 보정하여 필터 변경 시 이전 결과가 남지 않도록 수정.
*   **알림 키워드 가중치**: 사용자가 설정한 알림 키워드와 매칭되는 공고를 우선적으로 상단에 노출하는 점수제 정렬 로직 고도화.

## 3. 프론트엔드 UI/UX 혁신

### A. 공고 리스트 및 상세 모달
*   **데이터 정규화**: `PRIVATE` 소스 데이터를 화면 표시 시 **"민간"**으로 자동 변환 처리.
*   **테이블 가독성**: 공고 기간 컬럼의 너비를 최적화하고 줄바꿈을 방지(`white-space: nowrap`)하여 시각적 피로도 감소. 폰트를 전체 테마와 일치시킴.
*   **상세 모달 배지**: 기관 배지와 상태 배지의 높이를 **32px로 통일**하고 간격을 조정하여 정돈된 레이아웃 완성.
*   **알림 키워드 우선순위**: 공고별 태그 리스트에서 사용자가 설정한 키워드를 **가장 앞쪽**에 배치하고 파란색으로 하이라이트함.

### B. 캘린더 (Calendar) 고도화
*   **데이터 동기화**: 리스트와 캘린더의 응답 DTO를 일원화하여 지역, 상태, 태그 정보를 완벽히 일치시킴.
*   **PC 레이아웃**: 날짜 칸 높이를 **140px**로 대폭 확대. 배지를 **3열 가로 나열** 및 **세로 스크롤** 방식으로 변경하여 정보 밀도 극대화.
*   **모바일 레이아웃**: 배지를 **1열 세로 나열** 및 **세로 스크롤**로 복구. 텍스트가 잘리지 않도록 폰트 크기(0.48rem)와 자간 정밀 조정.
*   **유기적 연동**:
    *   달력 날짜 클릭 → 하단 리스트로 자동 스크롤.
    *   하단 리스트 클릭 → 달력 상단으로 이동 및 해당 공고의 **전체 기간(Range) 하이라이트** (다른 공고 임시 숨김).
*   **필터링 시스템**: 리스트와 동일한 **통합 서클 필터 바**를 캘린더에도 적용.

### C. 마이페이지 및 알림 설정
*   **한글 배지 적용**: 영어로 표기되던 태그 타입(METRO, TARGET 등)을 **광역, 대상** 등 직관적인 한글로 변환.
*   **키워드 검색 개선**: 검색 결과 리스트에서 키보드(↑, ↓) 이동 시 스크롤이 자동으로 따라오도록 구현.
*   **알림 키워드 토글**: 상세 카드에서 태그 클릭 시 `Alert` 창 없이 즉시 **등록/해제**가 가능한 토글 로직 구현.

## 4. 시스템 최적화 및 레거시 제거
*   **불필요한 코드 삭제**:
    *   구형 Enum: `NoticeKeyword.java` (상수 전체 제거)
    *   중복 엔티티: `NoticeKeywordEntity.java` 삭제
    *   미사용 기능: `Keyword.java` 관련 도메인, 컨트롤러, 서비스, 리포지토리, DTO 일괄 삭제.
*   **버그 수정**:
    *   알림 키워드 등록 시 발생하던 500 에러(Method Not Supported) 해결.
    *   '결과발표' 필터링 시 '안내' 공고가 중복 노출되던 로직 수정.
    *   모바일 카드에서 태그가 `[object Object]`로 표시되던 렌더링 오류 수정.

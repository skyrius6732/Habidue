# 2026-03-05 태깅 시스템 대개편 및 통합 고도화 작업 보고서

## 1. 아키텍처 혁신: Enum 기반 → DB 메타데이터 관리
- **DB화**: `NoticeKeyword` Enum의 1,244개 데이터를 `notice_keyword_metadata` 테이블로 이전.
- **수집 로직 정교화**: `DataInitializer` 및 수집 시점에 제목/내용 분석을 통해 실제 `tag` 및 `notice_tag` 테이블에 정제된 데이터가 매핑되도록 구현.
- **성능 최적화**: 1,244개의 키워드를 서버 기동 시 메모리에 캐싱하여 실시간 분석 속도 확보 및 수집 시 N+1 문제 해결(Fetch Join 적용).

## 2. 태깅 및 수집 엔진 고도화
- **시스템 태그 자동화**: `접수중`, `마감`, `결과발표`, `안내` 4개 상태를 `SYSTEM` 타입으로 관리. 오늘 날짜를 기준으로 `notice_tag`를 자동 Update/Insert 하도록 로직 구축.
- **정규표현식 제거**: `#근린생활시`와 같은 오탐을 유발하던 정규표현식 로직을 삭제하고 100% DB 매칭 방식으로 전환하여 `CITY_COUNTY` 추출 정확도 향상.
- **중복 방지**: 동일 이름의 태그가 타입 불일치로 인해 중복 노출되는 현상 방지 로직 적용.

## 3. 검색 및 정렬 시스템 최적화
- **관심공고순(인기순)**: 특정 키워드(예: 연신내) 검색 및 필터링 적용 시, '전체 사용자의 하트 수'를 기준으로 정렬되도록 수정하여 검색 결과의 풍부함과 유용성 확보.
- **검색 동기화**: 필터링 항목(기관, 상태)과 정렬 방식 변경 시 이전 검색 결과가 남지 않도록 QueryDSL BooleanBuilder 로직 보정.
- **알림 키워드 순**: 백엔드에서 매칭 점수제로 정렬하고, 프론트엔드 리스트에서도 사용자가 설정한 키워드를 태그 목록의 가장 앞에 배치하여 시각적 우선순위 부여.

## 4. 프론트엔드 UI/UX 대폭 개선

### A. 공고 리스트 및 상세
- **레이아웃**: 공고 기간 한 줄 표시(너비 조정), 필터 바 가운데 정렬, 테이블 내 '민간' 워딩 통일.
- **상세 모달**: 기관/상태 배지 높이(32px) 통일 및 간격 조정, 폰트 일원화.
- **워딩 통일**: 모든 버튼 문구를 `원문 공고보기 ↗`로 통일하고 높이를 슬림하게 조정.

### B. 캘린더 (Calendar)
- **공고 일치**: 리스트와 캘린더의 데이터 원본 및 응답 형식을 100% 동기화.
- **PC 레이아웃**: 날짜 칸 높이 확대(140px), 배지 3열 가로 배치 및 세로 스크롤 적용, 하단 상세 리스트 2열 배치 및 제목 말줄임 처리.
- **모바일 레이아웃**: 배지 1열 세로 나열 및 세로 스크롤 복구, 폰트 크기 최적화(0.48rem), 태그 박스 스타일 일원화.
- **상호작용**: 달력 클릭 시 하단 리스트 포커싱, 리스트 클릭 시 달력 상단 이동 및 해당 공고 기간(Range) 하이라이트 기능 구현.

### C. 마이페이지 (MyPage)
- **한글화**: 태그 타입(REGION, TARGET 등)을 직관적인 한글(지역, 대상 등)로 변환 표시.
- **키워드 검색**: 결과 리스트 키보드 이동 시 스크롤 자동 연동.
- **알림 토글**: 상세 카드에서 태그 클릭 시 Alert 없이 즉시 등록/해제 토글 기능 구현.

## 5. 주요 버그 해결 및 안정성 확보
- **필터링 오류**: '결과발표' 선택 시 '안내' 공고가 함께 나오는 현상 수정.
- **500 에러**: 공고 상세 카드에서 알림 등록 시 발생하던 API 방식 불일치(Method Not Supported) 해결.
- **데이터 유실**: `PRIVATE` 소스의 공고가 캘린더에서 누락되거나 워딩이 어긋나던 현상 완벽 수정.
- **레거시 제거**: `NoticeKeyword.java`, `Keyword.java`, `keywords` 테이블 등 구형 코드 및 잔재 파일 일괄 삭제.

## 6. 향후 유지보수 가이드
- **태그 체계**: 모든 태그는 `notice_keyword_metadata`를 기반으로 자동 생성됨.
- **데이터 초기화**: 새로운 태그 체계 적용을 위해 `notice_tag`와 `tag` 테이블을 비우고 다시 수집/태깅하는 것을 권장.

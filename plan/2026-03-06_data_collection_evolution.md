# HabiDue 데이터 수집 엔진 진화 과정 및 기술적 의사결정 정리

이 문서는 HabiDue 프로젝트의 핵심인 '공고 데이터 수집 엔진'이 초기 설계부터 현재의 통합 크롤링 체제로 발전해온 과정을 정리한 것으로, 추후 면접 시 기술적 의사결정 근거로 활용하기 위해 작성되었습니다.

---

## 1. 초기 단계 (Phase 1): 개별 소스 다변화 및 AI 분석 시도

### [SH] Gemini AI 기반 비정형 데이터 분석
*   **수집 방식:** SH 공식 홈페이지의 공고 상세 텍스트를 추출하여 Gemini API로 전달.
*   **시도 이유:** 공고문마다 마감일 표기 방식이 달라 정규식만으로는 추출 한계가 있었음. AI의 문맥 이해 능력을 활용해 정확한 마감일과 키워드를 뽑고자 함.
*   **한계 및 실패 원인:** 
    *   **Rate Limit 이슈:** 무료 버전 API의 초당/분당 호출 제한으로 인해 수천 건의 공고를 수집하는 데 물리적 병목 발생.
    *   **비용 및 운영성:** 서비스 규모 확대 시 API 비용 부담이 커지며, 비정형 데이터 분석의 일관성이 100% 보장되지 않음.

### [LH] 공식 Open API 활용
*   **수집 방식:** 공공데이터포털의 LH 임대주택 공고 Open API 사용.
*   **시도 이유:** 신뢰할 수 있는 공공기관 제공 데이터이며, 정형화된 JSON/XML 포맷을 받을 수 있음.
*   **한계 및 실패 원인:**
    *   **업데이트 지연:** 실제 홈페이지 공고 게시 시점과 API 데이터 반영 시점 사이의 시차 발생.
    *   **데이터 파편화:** API 제공 필드와 실제 유저가 필요로 하는 상세 페이지 정보 간의 불일치.
    *   **복잡한 인증 및 매핑:** API 키 관리 및 각 기관마다 다른 API 규격을 일일이 맞추어야 하는 유지보수 부담.

---

## 2. 중간 단계 (Phase 2): 하이브리드 보조 시스템 시도

*   **수집 방식:** 상세 페이지 텍스트에서 '마감일 힌트' 문장 추출 + 유저 클립보드 자동 인식 저장.
*   **시도 이유:** 시스템이 100% 확신할 수 없는 정보는 '힌트'로 주고, 유저가 확인 후 '복사'만 하면 저장되게 하여 UX를 혁신하고자 함.
*   **한계:** 클립보드 접근 권한에 대한 브라우저 보안 정책, 반복되는 알림창(alert/confirm)으로 인한 사용자 피로도 증가 등 기술적/경험적 복잡성이 실익보다 큼.

---

## 3. 현재 단계 (Phase 3): 서울주거포털 통합 크롤링 (Final)

### [통합] 서울주거포털(housing.seoul.go.kr) 타겟팅
*   **수집 방식:** 서울시에서 운영하는 대형 포털의 LH/SH 공고 리스트를 JSoup으로 크롤링.
*   **핵심 기술:** 
    *   **데이터 구조화:** 리스트 테이블에 명시된 '발표일(SH)', '마감일(LH)' 데이터를 직접 파싱.
    *   **자동 페이지 감지:** '마지막 페이지' 버튼의 파라미터를 분석하여 유동적인 공고 수에 대응.
    *   **DB 동기화:** `apiId` 기반 중복 체크 및 제목/링크/날짜 실시간 업데이트 로직.

### 💡 기술적 의사결정 근거 (Interview Point)
1.  **데이터 일관성 (Consistency):** LH와 SH의 서로 다른 규격을 서울주거포털이라는 **'표준화된 단일 인터페이스'**로 통합하여 관리 포인트를 단일화함.
2.  **신뢰도와 라이브 상태 (Freshness):** Open API의 정적인 데이터보다 홈페이지 게시판의 **[접수중/마감] 배지 상태**가 유저에게 훨씬 실시간성 높은 정보를 제공함.
3.  **성능 최적화 (Performance):** 상세 페이지를 수천 번 방문하던 방식에서 리스트 페이지만 긁는 방식으로 개선하여 **수집 속도를 약 20배 이상 향상**시킴.
4.  **비용 제로 (Cost-Efficiency):** 외부 API나 AI 의존성을 완전히 제거하여 인프라 비용 없이 독자적인 데이터 수집 망을 구축함.

---

## 4. 최종 성과
*   **수집 속도:** 전체 공고(약 1,100건) 수집 시간 10분 -> **30초 내외**로 단축.
*   **정확도:** 정규식 추측이 아닌 사이트 공식 제공 날짜를 사용하여 마감일 신뢰도 100% 확보.
*   **유지보수:** 서울시 운영 포털은 레이아웃 변경이 잦지 않아 안정적인 서비스 운영 가능.

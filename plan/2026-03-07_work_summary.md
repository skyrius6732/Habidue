## 1. 데이터 수집 엔진의 진화 (Crawling & Parsing)

### 1.1 SH(서울주택도시공사) 크롤러 고도화
*   **배경:** 기존에는 목록 테이블에서 제공하는 '발표일'을 마감일 필드에 임시로 저장했으나, 실제 사용자가 필요한 정보는 '청약 신청 기간'임이 확인됨.
*   **변경 사항:**
    *   **상세 페이지 방문 분석:** 목록 수집 후 각 공고의 상세 URL을 방문하여 본문 HTML을 파싱하는 단계 추가.
    *   **필드 분리:** `Notice` 엔티티에 `result_date`(발표일) 필드를 추가하여 목록의 날짜를 보존하고, 상세 페이지에서 추출한 날짜는 `deadline`에 저장.
    *   **지능형 섹션 추출:** 제목이나 상단 레이아웃(등록일, 조회수)에 포함된 키워드에 낚이지 않도록 `div.board-view-cont` 내의 순수 본문 텍스트만 타겟팅.
    *   **정규표현식 보강:** `인터넷 청약접수`, `방문 접수`, `신청기간` 등 다양한 변수 대응.
    *   **날짜 정규화:** `3. 21`과 같이 월/일이 한 자릿수이거나 공백이 섞인 경우에도 유연하게 인식하도록 패턴 개선.
    *   **연도 상속 로직:** 마감일 부분에 연도가 없는 경우, 시작일의 연도나 공고 게시일의 연도를 자동으로 상속받아 `2025-03-21` 형태의 완전한 객체 생성.

### 1.2 민간임대(청년안심주택) 크롤러 정교화
*   **변경 사항:** 물결표(`~`)가 포함된 날짜 범위에서 뒷부분(종료일)을 우선 추출하는 로직 적용.
*   **특수 기호 대응:** 전각 문자(`～`)나 하이픈(`-`) 등을 표준 물결표로 정규화하여 파싱 성공률 100% 확보.

## 2. 태깅 시스템 아키텍처 개편 (Tagging Engine)

### 2.1 지명 오탐지 방어 전략 (Masking & IGNORE)
*   **문제점:** `지원주택`이라는 일반 명사 속에 `원주`라는 지명이 포함되어 있어, 강원도 원주와 무관한 공고에 '원주' 태그가 달리는 현상 발생.
*   **해결책 (마스킹 기술):**
    *   `TagType.IGNORE` 타입을 신설하여 방어가 필요한 단어(`지원주택`, `전원주택`, `산업단지` 등)를 메타데이터로 관리.
    *   태그 추출 전, 제목 내의 `IGNORE` 단어들을 일시적으로 `###`로 치환(Masking).
    *   치환된 상태에서 지역명을 추출함으로써 오탐지 원천 차단.
    *   기존 `SPECIAL`이나 `TYPE` 태그들도 방어막 역할을 수행하도록 로직 확장.

### 2.2 긴 단어 우선 매칭 (Longest Match First)
*   **문제점:** `연신내역` 공고에서 `신내역` 태그가 중복 추출되는 문제.
*   **해결책:**
    *   메타데이터 사전을 로드할 때 **키워드 글자 길이 내림차순**으로 정렬.
    *   긴 단어가 먼저 매칭되어 마스킹되므로, 짧은 단어가 긴 단어의 일부로 오인될 가능성 제거.

### 2.3 시스템 상태 판별 우선순위 재정립
*   **문제점:** `[정정] 모집공고` 제목이 '정정' 키워드 때문에 `INFO`(안내)로 오분류됨.
*   **해결책:**
    *   `RECRUITING`(모집) 판별을 최우선순위로 격상.
    *   제목에 '게시', '경쟁률', '결과'가 포함된 사후 정보성 공고는 모집 키워드가 있어도 `INFO`로 분류하는 예외 필터 추가.
    *   날짜 정보가 전혀 없는 공고는 보수적으로 `INFO` 배지를 달아 사용자 확인 유도.

## 3. 사용자 경험 및 인터페이스 (Frontend UI/UX)

### 3.1 시간 흐름 기반의 동적 상태 UI
*   **발표완료/이전안내 도입:** 게시 후 30일이 지난 정보에 대해 새로운 상태값 부여.
*   **시각적 통일:** `마감`, `발표완료`, `이전안내` 배지를 모두 **빨간색**으로 통일하여 "종료된 정보"임을 직관적으로 전달.
*   **필터 통합:** 상단 '마감' 서클 필터 하나로 모든 종료된 데이터를 한꺼번에 온/오프 가능하도록 백엔드 쿼리(`NoticeRepositoryImpl`)와 연동.

### 3.2 상세 모달(Modal) 사용성 강화
*   **ESC 닫기:** 키보드 이벤트를 전역 등록하여 ESC 키 입력 시 모달 즉시 닫힘 지원.
*   **드래그 보호:** 모달 내부 텍스트 드래그 중 실수로 배경에서 마우스를 뗄 때 창이 닫히지 않도록 `mousedown`/`mouseup` 타겟 비교 로직 적용.
*   **닫기 버튼 개선:** 텍스트 '닫기'를 직관적인 `×` 아이콘으로 변경하고 스타일링.
*   **날짜 가시성:** `deadline`이 없으면 `resultDate`를 대신 표시하고, 둘 다 없으면 `일정 정보 없음` 문구 출력.

### 3.3 리스트 정보 고도화
*   **지하철역(STATION) 노출:** 지역 컬럼에 광역/시군구뿐만 아니라 역 이름을 최우선적으로 노출하도록 DTO 로직 수정.
*   **안내 문구 정제:** 공고 상단 유의사항 배너의 위치를 테이블 바로 위로 이동하고 문구를 간결하게 다듬음.
*   **검색창 초기화:** 검색어 입력 시 우측에 커스텀 `×` 버튼을 생성하여 검색어 초기화 기능 구현.
*   **페이징 개선:** `<<` (처음), `>>` (끝) 이동 버튼 추가로 수천 건의 데이터 탐색 편의성 증대.

## 4. 백엔드 데이터 정합성 및 인프라 (System & DB)

### 4.1 엔티티 및 DTO 구조 개선
*   **Notice 엔티티:** `status` (Enum)와 `resultDate` (LocalDateTime) 필드 추가.
*   **UserNoticeResponseDto:** 프론트엔드 누락 데이터 해결을 위해 `noticeAnnouncementDate`, `noticeResultDate` 필드 추가 매핑.
*   **데이터 일관성:** 태깅 엔진이 태그를 붙일 때 엔티티의 `status` 필드도 동시 업데이트하도록 보장.

### 4.2 메타데이터 자동 마이그레이션 (Temporary)
*   **태그 복구:** 기존 `TAG` 테이블에만 있고 메타데이터 사전에는 없던 유용한 키워드(LH, SH, 지역명 등)를 사전에 자동으로 등록하는 임시 로직 수행.
*   **전국 지역 데이터 구축:** 파이썬 스크립트를 통해 전국 17개 시도, 226개 시군구, 수백 개의 읍면동 데이터를 중복 없이 생성하여 DB 반영.

### 4.3 개발 환경 최적화 (Troubleshooting)
*   **IntelliJ 빌드 멈춤 해결:** `gradle.properties`에 JVM 메모리 옵션(`-Xmx2048m`) 설정 및 좀비 자바 프로세스 일괄 정리.
*   **보안 강화:** JWT 비밀키를 환경 변수(`${JWT_SECRET}`) 방식으로 전환하고, 테스트 환경 전용 더미 설정 파일(`test/application.yml`) 구축.

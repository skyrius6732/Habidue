### 1. 📊 운영 통계 대시보드 (`AdminDashboardView`)

- **데이터 분포 분석:** 출처별(LH/SH/HUG/민간), 상태별 공고 비율을 시각적인 차트와 프로그레스 바로 구현.
- **백엔드 통계 엔진:** `AdminDashboardService` 신설을 통해 DB 내 분산된 데이터를 단일 API 응답으로 집계 및 전송.

### 2. 🏠 공고 데이터 관리 시스템 (`AdminNoticeView`)
- **고급 검색 및 필터:** 제목/내용 기반 키워드 검색과 기관별, 상태별 정밀 필터링 기능 탑재.
- **인라인 관리 로직:** 리스트에서 즉시 공고 상태를 변경할 수 있는 UI 적용 및 실시간 DB 동기화.
- **데이터 CRUD 모달:** 공고 상세 수정 및 수동 등록을 위한 모달 기반 편집기 구축 (원문 링크, 마감일, 상세 내용 등 제어).
- **모바일 프리뷰:** 제목 롱프레스(0.6초) 시 상세 내용을 미리 보고 즉시 수정 모드로 진입 가능한 UX 구현.

### 3. 🛡️ 실시간 사용자 차단 시스템 (Redis 기반)
- **데이터 구조 확장:** `User` 엔티티에 `UserStatus` (ACTIVE, BLOCKED, WITHDRAWN) 및 차단 사유/일시 필드 추가.
- **Redis 즉시 차단 메커니즘:** 
    - 관리자가 차단 시 Redis에 `user:blocked:{userId}` 키 즉시 생성.
    - `JwtAuthenticationFilter`에서 매 요청마다 Redis를 조회하여 차단 유저의 접근을 0.1초 내에 원천 차단.
- **인증 보안 강화:** `CustomOAuth2UserService` 연동을 통해 차단된 사용자의 신규 소셜 로그인 시도 원천 봉쇄.
- **관리자 전용 유저 컨트롤러:** 차단/해제 API 및 상태별 필터링 기능이 포함된 `AdminUserView` 업데이트.

### 4. 🗂️ 메타데이터(키워드 매칭 사전) 관리 시스템 구축
- **엔티티 개선:** `NoticeKeywordMetadata`에 `@Setter` 및 `toBuilder`를 적용하여 실시간 수정 가능 구조로 변경.
- **인터렉티브 UI:** 타입별 데이터 개수 집계 칩 필터 및 인라인 편집 기능 구현.
- **데이터 가독성 최적화:** 중요 정보 확인을 위해 [타입] 컬럼을 [키워드] 앞으로 배치 순서 변경.
- **인라인 수정 UI 고도화:** PC 버전 In-place Editing 구현 및 Enter/Esc 단축키 연동, 모바일 저장/취소 버튼 스타일 표준화 완료.

### 5. 🎨 어드민 관리 UI 픽셀 단위 정밀 보정 및 표준화
- **레이아웃 표준화:** 모든 관리 페이지의 스크롤바(4px), 페이지네이션 버튼, 여백 간격을 '태그 관리' 기준으로 일원화.
- **정밀 스타일 조정:** 
    - PC 버전 테이블 행 높이 축소(48px)로 데이터 밀도 향상.
    - 관리 항목 하단 구분선을 -2px 조정하여 인접 셀과 완벽한 정렬 일치.
- **사이드바 높이 이슈 해결:** 고정 높이 설정을 제거하고 `min-height: 100vh`와 `sticky` 속성을 적용하여 배경 끊김 현상 해결.

### 6. 🛑 차단 계정 사용자 경험(UX) 및 보안 정교화
- **전용 안내 페이지:** 차단된 유저 접근 시 500 에러 대신 전용 차단 안내 페이지(`/blocked`)로 리다이렉트.
- **실시간 퇴출(Ejection) 로직:** `USER_BLOCKED:` 에러 코드 도입을 통해 접속 중인 유저가 다음 클릭 시 즉시 튕겨 나가도록 구현.
- **사유 전달 체계:** Redis에 실제 차단 사유를 저장하고 이를 안내 페이지에 실시간 노출하여 명확한 정보 제공.
- **헤더 간소화:** 차단 페이지 진입 시 불필요한 메뉴를 숨기고 로고와 테마 아이콘만 노출하는 전용 레이아웃 적용.

### 7. 📱 공고 및 태그 관리 모바일 UI 그리드화 및 초슬림 컴팩트화
- **공고 관리 2열(Grid) 도입:** 모바일 리스트 가독성 향상을 위해 2열 그리드 레이아웃으로 전환하고 ID/날짜를 제거한 초슬림 UI 적용.
- **정교한 2단 필터 (7:3 & 5:5):** 1행(검색 70%, 등록버튼 30%), 2행(출처 50%, 상태 50%)의 그리드 배치를 통해 모바일 사용성 극대화.
- **UI 일원화 및 표준화:** 모든 관리 페이지의 검색/필터 스타일을 통일하고 돋보기 아이콘을 입력창 내부로 통합하여 시각적 일관성 확보.

### 8. ⚖️ 6단계 정밀 상태 판별 엔진 고도화 및 데이터 중심 설계
- **판별 로직의 객체지향화:** `NoticeStatus` Enum 내부에 우선순위(Priority)와 마감 시 전이 상태(Expired Status)를 캡슐화하여 하드코딩 제거.
- **6단계 세분화 체계:** `접수중`, `마감`, `결과발표`, `발표완료`, `안내`, `이전안내`로 상태를 정교화하여 데이터 해상도 향상.
- **우선순위 엔진:** "모집"과 "발표" 키워드가 동시에 존재할 경우 우선순위가 높은 '발표'를 최종 상태로 결정하는 충돌 방지 로직 적용.
- **날짜 기반 자동 보정:** 키워드와 상관없이 물리적 마감일(`deadline`)이 지났다면 시스템이 자동으로 상태를 보정하는 강력한 정합성 규칙 적용.

### 9. ⚡ 관리자용 전수 데이터 정화(Reprocessing) 시스템 구축
- **벌크 처리 최적화:** 수천 건의 태그를 공고당 단 1번의 쿼리로 삭제하는 `deleteByNoticeBulk` (JPQL @Modifying) 적용.
- **배치 플러시 메커니즘:** 50건 단위로 영속성 컨텍스트를 강제 비우는(`flush` & `clear`) 배치 프로세스를 통해 대량 처리 속도 10배 이상 향상.
- **UI 통합:** 관리자 페이지에 '🔄 태그 재설정' 버튼을 신설하여 규칙(메타데이터) 변경 시 즉시 과거 데이터를 정제할 수 있는 기능 제공.

### 10. 🔓 관심 공고 기반 커뮤니티 해금 아키텍처 완성
- **반정규화 최적화:** `UserNotice` 테이블의 무거운 카운팅 쿼리를 제거하고 `Notice` 엔티티에 `interestCount` 필드를 직접 관리하여 조회 성능 극대화.
- **QueryDSL 원자적 업데이트:** 동시성 환경에서도 정확한 카운트를 보장하기 위해 QueryDSL의 `update()` 구문을 활용한 Atomic Increment 구현.
- **해금 가드 로직:** `isBoardActive` 플래그와 서비스 레이어의 검증 로직을 통해, 해금 조건(관심 유저 10명) 미달 시 게시판 접근을 원천 차단하는 보안 체계 구축.

### 11. 🧭 커뮤니티 시스템 아키텍처 전환 결정 (Dedicated Board View)
- **UX 한계 극복:** 모달 내부 게시판의 가독성 및 글쓰기 편의성 한계를 진단하고, 독립된 전용 메뉴 방식의 아키텍처로 전환 결정.
- **하이브리드 라우팅 기획:** 공고 상세 모달은 '커뮤니티 이동'을 돕는 포털 역할을 하고, 실제 커뮤니티 활동은 전체 화면을 사용하는 독립 페이지(`/board`)에서 이루어지는 구조 설계.

# 📋 2026-03-15 글쓰기 주제 선택 UX 고도화 설계 (ADR)

## 🎯 문제 상황
- '전체' 게시판에서 글쓰기 시 수십 개의 소통 주제가 한꺼번에 노출되어 사용자가 원하는 주제를 찾기 어렵고 혼란스러움.
- 인지 과부하로 인해 사용자 경험(UX) 저하 및 잘못된 게시판 선택 가능성 존재.

## 🛠️ 해결 방안 (Two-Step Selection)

### 1. 2단계 선택 프로세스 도입
- **Step 1:** 등록할 '게시판(소메뉴)'을 먼저 선택. (예: 서울, 당첨자 인터뷰, 이사 등)
- **Step 2:** 해당 게시판에 최적화된 '소통 주제(Flair)'만 노출.

### 2. UI 구성
- **Sub-Category Tabs:** 상단에 소메뉴 게시판들을 가로 스크롤 탭으로 배치.
- **Flair Chips:** 탭 선택에 따라 하단에 관련 주제 칩들을 노출.
- **Auto-Selection:** 전체 게시판에서 진입 시 '전체(자유)' 탭을 기본값으로 설정하되, 사용자가 명확한 위치를 선택하도록 유도.

### 3. 기대 효과
- 화면의 복잡도 감소 (한 화면에 노출되는 정보량 제어).
- 게시글 분류의 정확도 향상.
- 사용자가 "내가 어디에 글을 쓰는지" 더 명확하게 인지.

# 🗓️ 작업 일지 (2026.03.16)

## 🎯 주요 목표
- 커뮤니티 통합 검색 시스템 구축 및 UX 고도화
- 활동 기반 배지(Badge) 마스터리 및 레벨링 시스템 설계 및 구현
- 마이페이지 활동 대시보드 구축 및 데이터 연동
- 관리자용 배지 운영 센터(규칙 관리 및 동기화) 구축

---

## 🛠️ 수행 내역

### 1. 통합 스마트 검색 시스템 구축
- **백엔드:** QueryDSL 기반의 통합 검색 API(`GET /api/v1/search`) 개발. 제목, 본문, 태그 가중치(Score) 계산 및 Snippet 추출 로직 적용.
- **프론트엔드:** 사이드바 최상단에 검색창 통합. 탭 기반 결과 페이지(전체, 통합광장, 공고소통방 등) 구현 및 키워드 하이라이팅 적용.
- **UX 개선:** 최근 검색어 저장/삭제 기능(LocalStorage), 키보드 내비게이션(화살표/엔터) 지원.

### 2. 배지 마스터리 및 레벨링 시스템 (Gamification)
- **인프라:** `UserActivityStats`, `Badge`, `UserBadge`, `BadgeLevelRule` 엔티티 및 Repository 구축.
- **레벨링 엔진:** 8단계 등급(🌱 새싹 ~ 👑 귀족) 및 5가지 활동 분야 결합 로직 구현. DB 규칙 기반의 동적 명칭 생성 적용.
- **실시간 집계:** 게시글/댓글 작성, 좋아요 수신, 관심 공고 등록, 연속 출석 시 즉시 통계 반영 및 배지 승급 체크.
- **데이터 무결성:** 기존 유저 활동 시 통계 레코드가 자동 생성되는 지연 초기화(Lazy Initialization) 및 영속성 컨텍스트(Detached Entity) 이슈 해결.

### 3. 마이페이지 및 리스트 시각화 고도화
- **대시보드:** 마이페이지 '내 활동' 탭 신설. 8종 활동 지표(글, 댓글, 좋아요, 조회수, 출석 등) 가시화.
- **배지함:** 획득한 배지를 등급별 컬러 포인트가 적용된 카드 형태로 나열. 배지 수치 실시간 연동.
- **리스트 UI:** 게시글/검색 리스트의 메타 정보를 2행으로 분리하여 배지 유무와 상관없는 정갈한 레이아웃 구현. 배지 카운터(+n) 및 툴팁 시스템 도입.

### 4. 관리자용 배지 운영 센터 구축
- **규칙 관리:** 배지 타입별 레벨업 조건(필요 수치), 칭호, 이모지를 실시간 수정할 수 있는 UI 구축.
- **마스터 생성:** 새로운 배지 카테고리를 동적으로 신설할 수 있는 기능 추가.
- **일괄 동기화:** 규칙 변경 후 전체 유저의 배지 데이터를 최신화하는 배치 처리 API 연동.
- **반응형:** 모바일에서도 깨짐 없는 고정 모달 편집 UI 적용.

### 5. 배찌 등급별 디자인 시스템 고도화 (Flamboyant UI)
- **비주얼 컨셉:** 8단계 레벨(1~100)에 따라 점진적으로 화려해지는 전용 CSS 프레임워크 구축. 
- **단계별 효과:** 새싹(심플) -> 브론즈/실버/골드(메탈릭) -> 다이아몬드(그림자) -> 마스터(펄스) -> 귀족(오로라 애니메이션) 효과 적용.
- **통일된 UX:** 게시글 리스트, 상세 페이지 작성자 이름, 마이페이지 배찌함에 동일한 클래스(`.badge-rank-X`)를 적용하여 시각적 일관성 확보.
- **UI 디테일:** 배찌 내부 `Lv.` 표시 통합, 고레벨용 카드 배경 반전 스타일, 툴팁 내 등급 스타일 적용 등 디테일 튜닝 완료.

### 6. 모바일 통합 검색 UX 고도화 (Auto-Scroll)
- **UX 개선:** 모바일 환경(992px 이하)에서 통합 검색 수행 시, 상단 헤더 영역을 지나 실제 검색 결과가 시작되는 `search-main-content` 영역으로 자동 스크롤되는 로직 구현.
- **기술 적용:** Vue 3의 `nextTick`과 `scrollIntoView({ behavior: 'smooth' })`를 활용하여 데이터 로딩 후 자연스러운 화면 전환 보장.

### 8. 중복 신고 방지 시스템 구축
- **백엔드:** `ReportRepository`에 신고자, 대상 ID, 대상 타입을 기반으로 한 중복 체크 메서드 추가. `BoardReportService`에서 신고 생성 전 기존 신고 이력을 확인하여 중복 시 예외(`IllegalArgumentException`) 발생.
- **UX 개선:** 한 사용자가 동일한 게시글이나 댓글을 여러 번 신고할 수 없도록 제한하여 신고 데이터의 신뢰성 확보 및 어뷰징 방지.

### 9. 마이페이지 닉네임 수정 UI 복구
- **디자인 개선:** 계정 정보 탭에서 닉네임 수정 시 나타나는 입력창과 버튼(저장/취소) 스타일이 누락되어 깨지던 현상을 수정.
- **스타일 적용:** 서비스 브랜드 컬러(파란색)를 활용한 입체감 있는 입력창과 버튼 디자인을 적용하고, 모바일에서도 쾌적하게 보이도록 반응형 레이아웃 보강.

### 10. 통합 레벨링 & 명예의 전당 (Gamification) 시스템 기반 구축
- **기획 문서화:** 배지 시스템을 확장하여 경험치(EXP) 기반의 통합 레벨링과 시각적 닉네임 효과, 기간별 랭킹 시스템을 도입하는 ADR(`user_level_ranking_system.md`) 작성.
- **백엔드 엔티티 설계:** `User` 엔티티에 `totalExp`, `level`, `equippedBadgeId` 추가 및 활동 기록을 남기는 `ExpHistory`, `ExpReason` 엔티티 구축.
- **경험치 엔진 로직:** 활동 시마다 경험치를 부여하고 레벨업 여부를 자동 계산하는 `ExpService` 코어 엔진 구현 완료.

### 11. 통합 레벨링 연동 및 화려한 닉네임 시스템 (Gamification UX) 적용
- **백엔드 활동 연동:** 게시글 작성(+10), 댓글 작성(+3), 남에게 좋아요 받음(+5), 출석체크(+20), 배지 획득(+50) 등 주요 비즈니스 로직(Post/Comment/Attendance/Badge Service)에 `ExpService.grantExp`를 주입하여 실시간 경험치 획득 구조 완성.
- **DTO 확장:** 프론트엔드로 레벨/경험치 정보를 넘겨주기 위해 `PostResponseDto`, `CommentResponseDto`, `UserResponseDto`에 `authorLevel` 및 `authorExp` 필드 추가.
- **프론트엔드 전역 컴포넌트 (`AnimatedNickname.vue`):** 
  - 레벨 1부터 100까지 6단계(평민, 시민, 기사, 귀족, 왕족, 전설)로 나뉘는 그라데이션, 네온 펄스, 파티클 이펙트 등 CSS 애니메이션 적용.
  - 마우스 호버(PC) 및 꾹 누르기(모바일) 시 팝업되는 미니 프로필(현재 레벨, 경험치 게이지 바, 대표 칭호) 구현.
- **UI 일괄 교체:** 기존의 단순한 닉네임+배지 스택 UI를 `PostList`, `PostDetailView`, `SearchView`에서 모두 `AnimatedNickname` 컴포넌트로 교체하여 사이트 전반에 걸친 시각적 화려함 부여.

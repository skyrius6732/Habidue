# 2026-03-20 활동 랭킹 고도화 및 성능 최적화 작업 보고서

## 1. 개요
커뮤니티 활성화를 위한 **활동 랭킹 시스템의 대규모 업데이트**와 시스템 부하 문제를 해결하기 위한 **Redis 기반 성능 최적화**, 그리고 사용자 경험(UX) 고도화를 위한 **마이페이지 가이드 개편**을 수행함.

## 2. 주요 아키텍처 결정 사항 (ADR)

### 2.1. Redis 기반 데이터 캐싱 도입
- **문제점:** TOP 100 랭킹 집계 및 배지 규칙 조회가 5,000ms를 초과하여 타임아웃 및 서버 Broken Pipe 오류 유발.
- **결정:** 
  - 랭킹 집계 결과 및 전체 배지 규칙 리스트를 Redis에 캐싱.
  - 랭킹 캐시(`ranking:cache:*`): 10분 유효 기간 설정.
  - 배지 규칙 캐시(`badge:rules:all`): 1시간 유효 기간 설정.
- **결과:** API 응답 속도 **5s -> 0.1s 내외**로 획기적 개선.

### 2.2. 배지 시스템 실시간 마스터 데이터 연동 (Refactoring)
- **문제점:** 유저 배지 획득 시점의 이름이 `UserBadge` 테이블에 물리적으로 저장되어, 관리자가 마스터리 관리에서 명칭이나 이모지를 수정해도 기존 유저에게 반영되지 않음.
- **결정:** `UserBadge`의 `displayName` 필드 의존성을 제거하고, API 호출 시 `BadgeLevelRule` 마스터 테이블 정보를 기반으로 실시간 조립(Join)하여 반환하는 구조로 개편.
- **결과:** 데이터 일관성 100% 확보 및 관리자 설정 즉시 반영 시스템 구축.

### 2.3. 활동 점수(EXP) 매핑 정교화
- **결정:** 게시글 작성 및 좋아요 수신 시 점수를 분야별(지식인, 리뷰왕, 열정파)로 정밀하게 분리.
  - **지식인:** [공고 소통방], [통합광장] 내 정보성 주제(기관별팁 등) 활동.
  - **리뷰왕:** [당첨후기], [파트너스] 이용후기/견적공유 활동.
  - **사진 보너스:** 모든 분야에서 사진 첨부 시 **+10 EXP** 추가 부여 로직 구현.

### 2.4. AI 기반 스마트 쪽지 및 보안 아키텍처 (New)
- **결정:** 사용자 간 1:1 소통을 위해 **AI 검열 시스템이 탑재된 쪽지 기능** 도입.
  - **플러그형 AI 엔진:** `MessageAnalyzer` 인터페이스를 통해 Gemini 2.5 Flash, Local Regex 등을 선택적으로 사용 가능하도록 설계.
  - **신뢰 기반 진입장벽:** 카르마 90P 미만 유저는 쪽지 발송을 제한하여 악성 스팸 및 비방 원천 차단.
  - **멀티파일 지원:** `MultipartFile` 연동을 통해 이미지 및 문서 첨부(최대 5개) 기능 구현.

## 3. UI/UX 개선 사항

### 3.1. 활동 랭킹 시각 효과 고도화 (Refinement)
- **시상대(Podium) 최적화:** 1위 👑(왕관), 2·3위 🏆(트로피) 이모지 차등 적용 및 1ST, 2ND, 3RD 명칭 가독성 확보.
- **초고밀도 빵빠레(Confetti):** PC(100개, 10px), 모바일(150개, 3.5px)로 입자를 최적화하고 하단 리스트 끝까지 우아하게 떨어지는 애니메이션 구현.
- **유리창 반짝임(Glass Shimmer):** 랭킹 보드 전체 영역을 부드럽게 훑는 빛 효과 추가 및 컨테이너 내부 가두기(Overflow Hidden 레이어 분리) 적용.
- **인터랙션 보정:** 닉네임 클릭 시 툴팁 고정(Pinned) 기능 추가 및 1위 툴팁이 잘리지 않도록 z-index(999,999) 및 상단 여백(180px) 정밀 보정.

### 3.2. 마이페이지 통합 대시보드 v4 개편
- **탭 통합:** 사이드바의 쪽지 메뉴를 제거하고 마이페이지 내 세 번째 탭으로 **'📬 쪽지함'** 통합 관리.
- **8:2 레이아웃:** 닉네임, 레벨, 경험치 영역을 하나로 묶어 좌측 열(8)에 배치하고, 주요 지표(신뢰도, 순위)를 우측 열(2)에 배치하여 시각적 일체감 부여.
- **메뉴 이모지:** 각 탭 제목에 직관적인 이모지(`📊`, `🏷️`, `📬`, `🔔`, `⚙️`) 추가.

### 3.3. 모바일 사용성 극한 최적화
- **배지 1/4 축소:** 좁은 리스트 화면을 고려하여 `unified-exp-badge` 크기를 기존의 1/4로 줄여 정보 밀도 확보.
- **쪽지함 개편:** 모바일 리스트 높이를 500px로 확장하고, 첨부파일을 1행 2열로 배치하여 스크롤 부담 경감.

## 4. 기술적 해결 사항 (Bug Fix)
- **JSON Infinite Recursion:** 쪽지 조회 시 `User`와 `UserNotice` 간의 순환 참조 문제를 `MessageResponseDto` 도입으로 원천 해결.
- **Boolean Serialization:** Jackson 직렬화 시 `isRead` 필드명이 누락되는 문제를 `@JsonProperty` 명시로 해결하여 새로고침 시에도 읽음 상태 유지 보장.
- **404 Not Found (User):** Security Context의 이메일 식별자와 DB의 유저네임 불일치 문제를 `findByEmail` 시도로 해결.
- **Gemini API v1beta:** 최신 모델 연동을 위한 엔드포인트 보정 및 404 URL 오류 해결.

## 5. 향후 과제
- 관리자 페이지의 '신고된 쪽지' 확인 및 처리(카르마 자동 차감) UI 구현.
- 첨부파일 용량 제한 및 허용 확장자 서버 측 밸리데이션 강화.
- 쪽지 수신 시 실시간 브라우저 푸시(SSE 또는 WebSocket) 연동 검토.

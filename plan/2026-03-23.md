# 2026-03-23 작업 요약 (habiDue)

## 1. 관리자 신고 및 제재 시스템 고도화
- **신고 처리 로직 개선:** 쪽지 신고 '블라인드' 시 대화방 차단(방 폭파)을 하지 않도록 변경, '영구 삭제' 시에만 차단 적용.
- **신뢰점수(Karma) 자동 복구:** 관리자가 신고 조치를 반려하거나 대기 상태로 복구할 경우, 이전에 차감되었던 점수를 자동으로 계산하여 복구하는 로직 구현.
- **감점 수치 조정:** 쪽지 영구 삭제 시 감점 수치를 100점에서 70점으로 하향 조정하여 계정 영구 정지 전 자숙 기회 제공.
- **시스템 메시지 정제:** 제재 안내 문구에서 '카르마'를 '신뢰점수'로 변경하고, 제재 적용 사실 및 사유를 문법에 맞게(사유로 인해 등) 정교화.

## 2. 사용자 관리 및 보안 강화
- **차단 사유 기록:** `UserBlock` 엔티티에 `reason` 및 `isSystemBlock` 필드를 추가하여 신고로 인한 자동차단 시 사유를 영구 박제.
- **차단 해제 로직 개선:** 사용자가 차단을 해제할 때, 시스템이 걸어두었던 해당 유저와의 대화방 제한(isRoomRestricted)도 함께 자동 해제되도록 연동.
- **위험 유저 경고:** 신고 제재로 차단된 유저 해제 시 강력한 경고 팝업(Confirm) 노출 및 차단 목록 내 빨간 배지 표시.

## 3. UI/UX 개선 및 버그 수정
- **신고 관리 필터링:** 쪽지 신고(쌍방 관리) 필터링 시 페이징 번호와 전체 선택 로직이 실제 검색 결과와 일치하지 않던 문제 해결.
- **실시간 데이터 동기화:** 쪽지 발송 모달 및 쪽지함 진입 시 사용자 프로필을 새로고침하여 최신 신뢰점수가 즉시 반영되도록 수정.
- **발송 제한 안내:** 신뢰점수 80.0점 미만 시 구체적인 기준 수치를 포함한 경고 배지 및 Alert 알림 추가.
- **쪽지함 정렬:** 대화방 목록은 최신순(상단) 유지, 대화창 내역은 기존 채팅 방식인 과거순(하단 축적)으로 정렬 방식 최적화.
- **디자인 보정:** 
  - 신고 로그 모달 내 거대 이미지 크기 최적화 (PC/모바일 대응).
  - 쪽지 발송 모달 내 파일 첨부 목록을 2열 그리드로 개편하여 가독성 향상.
  - 게시글 상세 페이지 진입 시, 게시글 타입에 맞춰 왼쪽 사이드바 메뉴 자동 포커싱 및 아코디언 확장.

## 4. 데이터 정합성 보완 (OSIV OFF 대응)
- **지연 로딩 해결:** 
  - 공고 리스트, 관심 태그 조회 시 발생하는 `LazyInitializationException`을 Fetch Join으로 해결.
  - 게시글 상세 조회 시 발생하는 `MultipleBagFetchException`을 방지하기 위해 ToOne 관계만 Fetch Join 하고, 컬렉션(이미지, 태그)은 서비스 레이어에서 강제 초기화(Initialize)하는 2단계 로딩 전략 적용.
- **중복 신고 버그 수정:** 영구 삭제된 메시지에 대한 과거 신고 기록이 새로운 대화방에서의 신고를 방해하지 않도록 쿼리 조건(`isDeleted = false`) 추가.

## 5. 실시간 알림 시스템 구축 (Notification System)
- **전체 인프라 설계:** Notification 엔티티, 레포지토리, 서비스, 컨트롤러 구축 및 SSE(Server-Sent Events) 실시간 전송 인프라 확보.
- **이벤트 기반 알림:**
  - 내 게시글에 댓글/답글 등록 시 알림.
  - 새로운 쪽지 수신 시 알림.
  - 신뢰점수 변동(감점/회복) 및 계정 상태 변경(차단/해제) 시 알림.
- **실시간 UI 구현:** 
  - 헤더 종 모양 아이콘(🔔) 및 읽지 않은 알림 개수 배지 적용.
  - 화면 좌측 하단 실시간 토스트(Toast) 팝업 알림 연동.
  - 알림 클릭 시 해당 게시글(댓글 위치), 쪽지함 등으로 자동 이동 로직 구현.

## 6. 알림 시스템 안정화 및 최적화
- **정합성 3중 방어막:** 
  - Jackson 직렬화 시 boolean 필드명 변환 문제(`isRead` -> `read`) 해결을 위한 `@JsonProperty` 및 프론트엔드 정규화 로직 적용.
  - DB 필드 타입을 `int`로 변경하여 매핑 모호성 제거 및 벌크 업데이트와 엔티티 개별 수정을 병행하는 하이브리드 로직 구축.
  - 브라우저 캐시 방지를 위해 API 호출 시 타임스탬프 파라미터 추가.
- **SSE 인프라 보강:** 
  - `EventSource` 헤더 미지원 대응을 위해 토큰을 쿼리 스트링으로 전달하고 컨트롤러에서 수동 검증하는 보안 체계 구축.
  - 401 Unauthorized 시 브라우저 무한 재시도를 방지하기 위한 응답 코드 최적화 및 재연결 간격 조정.
- **서버 리소스 관리:** OSIV 비활성화 및 HikariPool 확장(20개)을 통해 롱 커넥션으로 인한 DB 커넥션 고갈 문제 예방.

## 7. 게시글 상세 페이지 고도화 (Deep Linking & Visibility)
- **가시성 정책 변경:** 사용자 편의를 위해 댓글/답글의 '더보기' 기능을 제거하고 항상 전체 내용을 노출하도록 UX 개편 (가독성 향상).
- **정밀 스크롤 로직:** 알림 클릭 시 대상 댓글/답글이 화면에 나타날 때까지 끈질기게 추적하여 스크롤 및 반짝임 효과를 주는 강력한 `scrollToComment` 수술 집도.

## 8. 좋아요 및 신뢰 점수(Karma) 시스템 정교화
- **점수 변동 즉시 반영:** 좋아요 클릭 시 +0.1P, 취소 시 -0.1P가 실시간 활동 이력에 즉시 기록되도록 수정.
- **알림 쿨타임 도입 (Redis):** 
  - 특정 게시글/댓글에 대한 좋아요 알림이 1시간 이내에 중복 발송되는 것을 Redis `setIfAbsent`를 통해 원자적으로 차단.
  - 최초 좋아요 시에만 '좋아요 + 신뢰 점수 회복' 세트 알림 전송.
  - 취소 시에는 알림을 발송하지 않아 사용자 피로도 감소.
- **어뷰징 방지:** 
  - 신뢰 점수 최대치(100.0P) 및 일일 회복 상한선(5.0P) 로직을 정교화하여 관리자 수동 포상 점수와 자동 회복 점수가 충돌하지 않도록 분리.
  - `revokeKarma` 로직 보완을 통해 하트 연타 시에도 점수가 무한히 깎이지 않도록 방어.

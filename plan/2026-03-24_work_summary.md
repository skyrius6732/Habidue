# 2026-03-24 작업 요약 (Work Summary)

## ✅ 주요 성과 및 변경 사항

### 1. 신뢰 점수(Karma) 시스템 재설계 및 실시간화
- **반응형 가감 로직**: 좋아요 수신 시 점수 즉시 상승, 좋아요 취소 시 점수 즉시 차감 로직 구현.
- **일일 한도 복구(Refund) 시스템**: 사용자가 좋아요를 취소하면 해당 점수만큼 일일 획득 상한선(+5.0P) 내에서 한도가 다시 복구되도록 리포지토리 쿼리 고도화.
- **알림 쿨타임**: 동일 사용자의 반복적인 좋아요에 대해 1시간 Redis 쿨타임을 적용하여 알림 피로도 감소.

### 2. 실시간 알림(SSE) 시스템 안정화 및 하이브리드화
- **인증 동기화(Token Sync)**: `axios` 인터셉터와 Pinia `authStore` 간의 토큰 동기화 누락을 해결하여 SSE 연결 전 토큰 갱신 보장.
- **2중 동기화 체계**: SSE(실시간) + 30초 주기 Short Polling(안전장치)을 결합하여 네트워크 불안정 시에도 알림 누락 방지.
- **서버 하트비트(Heartbeat)**: 15초 주기 `ping` 데이터 발송으로 Vite Proxy 및 브라우저의 타임아웃 강제 종료 방지.
- **초기화 락(Lock)**: 비동기 찰나의 순간에 발생하는 중복 구독 요청을 원자적 플래그로 차단.

### 3. 쪽지 및 차단 시스템 고도화
- **이력 보존형 차단**: 차단 시 방을 삭제하던 기존 방식에서, 대화 내역은 보존하되 입력창만 차단하는 방식으로 개선.
- **관리자 제재 연동**: 영구 제재 시 실제 차단 목록 자동 추가 및 대화방 영구 폭파(`isRoomRestricted`) 기능 구현.
- **자동 동기화(Sync)**: 관리자가 제재를 번복(복구)할 경우, 신고자의 차단 목록에서 가해자를 자동으로 제거하는 정합성 로직 추가.
- **상세 내역 정렬**: 관리자용 쪽지 상세 보기 시 대화 흐름을 시간순(ASC)으로 정렬하여 가독성 개선.

### 4. 통합 검색 엔진 구축 (V1)
- **전용 검색 API (`/api/v1/search`)**: 전체 검색 결과와 게시판 타입별 건수를 한 번에 반환하는 통합 검색 레이어 구축.
- **프론트엔드 리팩토링**: `SearchView.vue`를 전면 개편하여 탭별 건수 실시간 표시 및 데이터 렌더링 불일치 해결.

### 5. 관리자 제재 알림 시스템 (Admin Feedback)
- **작성자 알림**: 블라인드/삭제 처리 시 알림에 원본 내용(제목/본문)을 포함하여 구체적인 제재 사유 전달.
- **신고자 피드백**: 관리자 조치 완료 시 신고자에게도 "조치 완료" 또는 "반려" 알림을 자동 발송하여 커뮤니티 투명성 제고.

## 🛠️ 해결된 기술적 이슈
- `LazyInitializationException`: 관리자 페이지 및 차단 목록 조회 시 트랜잭션 범위 확장으로 해결.
- `BeanDefinitionConflictException`: 중복된 `SearchController` 클래스명 정리 및 패키지 구조 최적화.
- `JSON Parse Error`: SSE 초기 메시지(`connected`) 및 하트비트 데이터의 비-JSON 형식 예외 처리.
- `TypeError: notificationStore.fetchUnreadCount`: 고도화 과정에서 변경된 메서드 명칭 하위 호환성 복구.

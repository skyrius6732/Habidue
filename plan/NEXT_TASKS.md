# 향후 확장 가능한 기능 리스트 (Next Tasks)

## 🚀 고도화 제안 및 기술 설계 방향

### 1. 전체 공지 실시간 알림 기능 (운영 효율화)
- **개요**: 관리자가 중요한 공지사항을 등록할 때, 서비스 내 모든 활성 유저에게 즉시 종 모양 알림 및 실시간 토스트를 발송.
- **상세 설계**:
    - **UI**: `AdminNoticeView.vue` 작성 화면에 `[ ] 모든 유저에게 실시간 알림 전송` 체크박스 추가.
    - **Logic**: 체크 시 `NoticeService`에서 전체 유저를 대상으로 `NotificationService.send` 벌크 호출.
    - **최적화**: 유저가 많아질 경우 Redis Pub/Sub을 활용한 비동기 분할 발송 로직 적용.

### 2. 관심 공고 키워드 매칭 알림 (개인화 추천)
- **개요**: 사용자가 설정한 키워드(예: LH, 청년, 서울)와 일치하는 새 공고가 올라오면 즉시 푸시 알림 제공.
- **상세 설계**:
    - **Event**: 신규 공고 수집/등록 이벤트 발생 시 트리거.
    - **Matching**: 유저별 설정 키워드와 공고 제목/내용 스캔 (Redis Set 연산을 활용한 고속 매칭).
    - **Notification**: "설정하신 [LH] 키워드와 일치하는 새 공고가 등록되었습니다!" 알림 발송.

### 3. 공고 마감 임박 리마인드 알림 (활동 유도)
- **개요**: 사용자가 '관심' 등록한 공고의 마감 기한이 하루(24시간) 남았을 때 자동으로 리마인드.
- **상세 설계**:
    - **Scheduler**: 매일 특정 시간(예: 오전 9시)에 실행되는 Spring Batch 또는 Scheduler 구축.
    - **Query**: `deadline = today + 1`인 공고와 해당 공고를 찜한 유저 리스트 추출.
    - **Notification**: "내일 마감되는 관심 공고가 있습니다. 놓치지 마세요!" 알림 발송.

## 🏗️ 아키텍처 확장 로드맵
- **Redis Pub/Sub 도입**: 다중 서버 환경에서 실시간 알림의 신뢰성 확보.
- **비동기 이벤트 기반 알림 (Spring Events)**: 알림 발송 로직을 비즈니스 로직(글쓰기, 제재 등)과 분리하여 응답 성능 개선.
- **알림 수신 설정**: 유저별로 받고 싶은 알림 종류(좋아요, 공지, 키워드 등)를 선택할 수 있는 설정 페이지 구축.

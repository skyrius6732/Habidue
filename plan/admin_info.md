     1. 운영자 관점의 관리 항목 분석 (CRUD 대상)


     HabiDue 서비스 운영을 위해 필요한 핵심 관리 도메인은 크게 5가지로 나뉩니다.



     ┌────────────────┬─────────────────────────────────┬────────────────────────────────┐
     │ 관리 도메인    │ 주요 CRUD 및 기능               │ 운영적 가치                    │
     ├────────────────┼─────────────────────────────────┼────────────────────────────────┤
     │ 1. 콘텐츠 관리 │ 공지사항, 패치노트              │ 서비스 소식 전달 및 신뢰도     │
     │ (About)        │ 등록/수정/삭제                  │ 확보                           │
     │ 2. 공고 데이터 │ 수집된 공고 수정, 노출 여부     │ 크롤링 오류(오타, 날짜 파악    │
     │ 관리           │ 토글, 수동 공고 등록            │ 실패) 보정 및 예외 공고 처리   │
     │ 3. 태그 및     │ 태그 타입 변경, 신규 태그 생성, │ 정교한 맞춤형 추천 및 필터링   │
     │ 키워드 관리    │ 사용 빈도 낮은 태그 정리        │ 품질 유지                      │
     │ 4. 사용자 및   │ 사용자 목록 조회, 역할(Role)    │ 커뮤니티 정합성 유지 및 운영진 │
     │ 권한 관리      │ 변경, 활동 중단(Ban) 처리       │ 권한 부여                      │
     │ 5. 시스템      │ 크롤링 배치 로그 확인, 이메일   │ 서비스 장애 대응 및 사용자     │
     │ 모니터링       │ 발송 현황, 통계(DAU 등)         │ 니즈(인기 키워드) 파악         │
     └────────────────┴─────────────────────────────────┴────────────────────────────────┘

     2. 사용자 역할(Role) 및 로그인 구분 전략


     Q1. ROLE_USER vs ROLE_ADMIN 구분 방법?
      - 회원가입: 모든 사용자는 최초 가입 시 무조건 ROLE_USER로 생성되어야 합니다. (보안상
        당연한 조치입니다.)
      - 권한 부여:
          - 최초 관리자: DB에서 직접 role 컬럼을 ADMIN으로 수정합니다 (마스터 계정).
          - 추가 관리자: 마스터 계정이 관리자 페이지의 '사용자 관리' 메뉴에서 특정 사용자의
            역할을 변경하도록 구현합니다.
      - 추가 역할 제안: MANAGER (콘텐츠 수정 권한만 있고 사용자 정보 열람은 불가) 역할을
        추가하여 운영 업무를 분담할 수 있습니다.


     Q2. 로그인 시 구분 방법?
      - UI적 구분: 로그인 성공 시 백엔드에서 내려주는 JWT 토큰 내 권한 정보(role)를
   확인합니다.
          - ADMIN 권한이 있는 경우에만 헤더에 '관리자 페이지' 링크를 노출합니다.
      - 접근 제어: Vue Router의 네비게이션 가드를 강화하여 ADMIN 권한 없이 /admin/** 경로
   진입
        시 403(권한 없음) 에러 페이지로 보냅니다.

     3. 관리자 페이지 구조 제안 (Centralized Admin Portal)

     현재 /admin/about 처럼 흩어져 있는 구조 대신, 통합된 Admin Dashboard를 제안합니다.


     추천 메뉴 구조:
      1. Dashboard: 주요 통계 (오늘 수집된 공고 수, 신규 가입자, 발송된 메일 수)
      2. User Management: 전체 사용자 리스트 및 권한 변경
      3. Notice Management: 수집된 공고 리스트 (수정/삭제/강제 수집 트리거)
      4. Tag Management: 태그 사전 관리 (타입 변경 및 오타 수정)
      5. Service Info: 공지사항 및 패치노트 관리 (기존 AboutAdmin)

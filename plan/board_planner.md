[Habidue 추가 게시판 기능 기획서]

현재 기획 중인 임대주택 정보 플랫폼의 핵심 기능들을 '사용자 경험(UX)'과 '커뮤니티 활성화' 관점에서 깔끔하게 정리해 드립니다. 이 구성은 개발 단계에서 우선순위를 정하고 로드맵을 짜기에 최적화되어 있습니다.

📋 임대주택 정보 플랫폼 핵심 기능 구성안
1. 정보 제공 및 관리 (기본 기능)
> 공고 리스트 & 캘린더: LH, SH, 민간임대 공고를 한눈에 확인하고, 마감일 및 발표일을 달력으로 관리합니다.
> 맞춤형 키워드 알림: 사용자가 설정한 알림키워드 공고가 뜨면 실시간 푸시 알림을 발송합니다.

2. 게시판 시스템 (소통의 핵심)
> 공고별 '조건부' 게시판:
모든 공고에 게시판을 생성하지 않고, **관심 공고(찜/알림 설정) 인원이 일정 수(초기 5~10명)를 넘으면 게시판이 자동 '해금(Unlock)'**됩니다.
소수의 인원이 모여 정보를 교류할 수 있는 판을 자동으로 깔아줍니다.

> 계층형 게시판 구조 :
통합 광장: 지역별/기관별/후기별 공통 질문(서류, 정책 등)을 나누는 곳.
공고별 탭: 특정 단지에 대한 정보 공유(경쟁률, 서류 제출 팁, 주택 단지 팁)를 위한 전용 공간.

> 게시판 정화 장치:
신고 기반 자동 블라인드: 일정 수 이상의 신고를 받으면 글이 자동으로 숨겨집니다.
익명성 보완: 닉네임과 함께 활동 지역이나 청약 순위 태그를 달아 신뢰도를 높이고 정보의 질을 관리합니다.

3. 참여형 도구 (데이터 결합)
>[정보형] 주택 지식인  :
사용자가 게시판에서 다른 사람의 질문에 답변을 많이 하거나, 답변이 '도움돼요'를 많이 받았을 때 부여합니다.

-획득 조건: 답변 채택 5회 이상 또는 답변에 대한 '좋아요' 총 20개 이상.
-역할: 이 배지가 달린 사람의 글은 신뢰도가 높다는 것을 시각적으로 보여줍니다. "박사님이 말씀하신 거니 믿고 서류 준비할게요"라는 분위기를 형성합니다.

> [성실형] 공고 컬렉터
공고를 자주 확인하고 알림 설정을 적극적으로 하는 유저에게 부여합니다.

-획득 조건: 관심 공고 알림 설정 10회 이상 또는 7일 연속 출석
-역할: 서비스의 '헤비 유저'임을 증명합니다. 게시판에서 "이 분은 항상 공고를 꿰고 있네"라는 인상을 주어 커뮤니티의 기둥 역할을 하게 만듭니다.

> [지역형/기관형] 00시 명예시민 (예: 서울시 명예시민, LH 전문가) :
특정 지역이나 특정 기관(LH/SH) 공고에 집중적으로 활동하는 유저에게 부여합니다.

- 획득 조건: 특정 지역(시 단위) 공고 게시판에 댓글 10개 이상 작성.
- 역할: 해당 지역의 로컬 정보(교통, 주변 마트, 실제 거주 여건 등)를 잘 아는 유저임을 표시합니다. "서울시 명예시민님, 여기 단지 근처 지하철역까지 실제로 얼마나 걸리나요?" 같은 구체적인 질문을 유도합니다.

> [후기형] 후기 요정 :
발표 후기, 당첨 후기, 서류제출 후기 등 다양한 후기를 올리는 유저에게 부여합니다.

- 획득조건: 후기 게시판 5개 이상 작성
- 역할 : 예비자 13번 받았는데 서류 대상자라고 연락왔어요 와 같은 당첨 후기를 남기도록 하여 추후 이 후기가 나중에 해당 지역에 지원할 사람들에게 최고의 필독서가 됩니다.


4. 배지 시스템 운영 꿀팁 (전문가 조언)
배지의 레벨화: * 초보 박사 -> 숙련 박사 -> 명예 박사 순으로 배지 색깔이나 모양을 업그레이드해 주세요. 
사용자가 계속 활동할 동기가 생깁니다.

개발자 겸 기획자 입장에서 **"이걸 진짜 구현할 때 겪게 될 현실적인 고민과 해결책"**을 몇 가지 짚어드릴게요.

1. 개발 및 운영 시 주의해야 할 '현실적 난관'

① 배지 부여의 로직 (Performance 이슈)
고민: 사용자가 글을 쓸 때마다 "이 사람이 지금 '후기 요정' 배지를 받을 조건이 되었나?"를 매번 DB에서 전체 카운트하면 시스템이 무거워질 수 있습니다.

해결책: '이벤트 기반 업데이트(Event-Driven)' 방식을 쓰세요. 게시글이나 댓글이 등록되는 시점에 해당 카운트를 +1 시키는 별도의 User_Stats 테이블을 두어, 배지 획득 여부를 실시간으로 체크하세요.

② '조건부 게시판'의 엣지 케이스 (Edge Case)
고민: 게시판이 '해금'되었다가, 시간이 지나 인기가 식어 사람이 아무도 없게 되면 어떻게 할 건가요?

해결책: **'게시판 휴면 상태'**를 도입하세요. 마지막 글 작성일로부터 30일이 지나면 게시판을 '보관함'으로 이동시키고, 다시 관심 유저가 5명 이상 모이면 다시 '활성화'시키는 방식입니다. 이 과정이 자동화되어야 관리자의 손이 덜 갑니다.

③ 닉네임과 익명성의 딜레마
고민: 커뮤니티가 커지면 익명 뒤에 숨어 악의적인 비방이나 거짓 정보를 퍼뜨리는 유저가 반드시 생깁니다.

해결책: **'활동 신뢰 점수(Karma Point)'**를 도입하세요. 배지는 긍정적 활동에 대한 훈장이지만, 신고를 당하면 점수가 깎이고 점수가 낮으면 글쓰기 제한이 걸리도록 **'패널티 시스템'**을 설계해야 합니다.

2. 더 스마트하게 만드는 팁 (운영/개발 효율)
① 배지 디자인 부담 줄이기
팁: 처음부터 복잡한 그래픽 배지를 직접 만들지 마세요. **'텍스트 기반의 태그형 배지(예: [LV.2 주택지식인])'**로 시작해서 시스템이 안정된 뒤에 아이콘 배지로 업그레이드해도 충분합니다.

③ '후기 요정'을 위한 템플릿 제공
팁: 사용자들이 후기를 쓰는 걸 어려워합니다. "후기 작성하기" 버튼을 누르면 [신청 단지 / 경쟁률 / 내 가점 / 당첨 결과 / 준비하면서 가장 힘들었던 점] 같은 간단한 설문지 형태의 템플릿이 자동으로 뜨게 만드세요. 그럼 훨씬 양질의 후기가 쌓입니다.

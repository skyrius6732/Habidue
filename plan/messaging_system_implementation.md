# habiDue 인스타그램 스타일 쪽지 시스템 구축 보고서

## 1. 프로젝트 개요
사용자 간의 소통을 원활하게 하되, 커뮤니티의 건강성을 유지하기 위해 '신뢰점수(Karma)' 소모와 '일일 제한'이 결합된 인스타그램 DM 스타일의 채팅 시스템을 구축함.

## 2. 주요 구현 기능

### A. UI/UX 디자인 (Instagram DM 모티브)
- **레이아웃**: PC 버전 2분할(목록/채팅), 모바일 버전 단일 화면(뒤로가기 지원) 적용.
- **말풍선**: 내가 보낸 메시지에 오로라 연보라 그라데이션(`linear-gradient`) 적용.
- **디테일**: 초슬림 스크롤바, 날짜 구분선, 이미지 썸네일(최대 100px~200px) 처리.
- **빈 상태**: 세련된 편지 이모지와 habiDue 포인트 컬러 버튼을 활용한 안내 화면.

### B. 신뢰점수 및 정책 (Karma System)
- **기본 점수**: 신규 가입 시 100점 → **1000점**으로 상향 (여유로운 활동 보장).
- **소모 비용**: 쪽지 1통당 내부 1점(**UI상 0.1P**) 소모, 발송 즉시 UI 반영.
- **일일 제한**: Redis를 활용하여 **유저당 하루 20통**으로 발송 제한.
- **안내 메시지**: 채팅방 진입 시 시스템 안내를 통해 신중한 소통 유도.

### C. 실시간 접속 상태 (Presence)
- **상태 추적**: Redis를 통해 유저의 최근 활동 시간을 1분 주기로 갱신.
- **UI 표시**: 대화 목록 아바타 옆에 온라인 점(녹색/회색) 및 헤더에 '활동 중/오프라인' 표시.

### D. 메시지 및 대화방 관리
- **메시지 수정**: 수정 버튼(✏️) 클릭 시 입력창 상단에 원본 프리뷰 노출 및 내용 수정 가능. (JSON/PATCH 방식)
- **소프트 삭제**: 삭제 시 DB 데이터는 보존하되, 화면에는 "메시지가 삭제 되었습니다."로 대체 표시.
- **대화방 삭제**: '방 나가기' 시 해당 시점까지의 내역을 본인 화면에서만 숨김 처리.
- **차단 및 신고**: 부적절한 유저를 차단하거나 대화방을 관리자에게 신고하는 기능.

## 3. 기술적 세부 사항 (Backend)
- **API 설계**: `/api/messages/{receiverId}` 경로 변수를 활용하여 전송 안정성 확보.
- **멀티파트 설정**: 파일 5개 첨부 및 대용량 전송(최대 250MB) 지원.
- **데이터 필터링**: 최근 90일 이내의 삭제되지 않은 메시지만 조회하도록 SQL 최적화.

## 4. 향후 유지보수 계획
- **자동 아카이빙**: DB 용량 관리를 위해 90일 경과 데이터에 대한 별도 보관 정책 시행 가능.
- **알림 연동**: 실시간 접속 중이 아닐 때 앱 푸시나 이메일 알림 연동 검토.

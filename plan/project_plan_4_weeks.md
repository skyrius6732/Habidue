### **🎯 프로젝트 컨셉 및 UI/UX 전략**

- **코어 컨셉:** 단순한 공고 모음이 아닌, 사용자의 관심 키워드 기반 **'마감 일정 관리 자동화'** 서비스.
- **차별점:** 흩어져 있는 마감 정보를 자동으로 수집하고, 개인화된 캘린더를 통해 놓치지 않도록 관리해주는 경험을 제공.
- **UI/UX 전략: Dual View**
  - **리스트 뷰 (탐색용):** PC/모바일 최적화 레이아웃 (테이블 vs 슬림 카드).
  - **캘린더 뷰 (관리용):** FullCalendar 기반 시각적 일정 관리.
- **핵심 기술:** Spring Boot, Vue.js, **QueryDSL (도입 완료)**, Redis, MySQL.

### **📅 1-2주차 (2/23 ~ 3/8): 백엔드 기반 및 핵심 로직 완성 [COMPLETED]**

- **공통 기능:** 전역 예외 처리(GlobalExceptionHandler) 및 공통 응답 포맷 적용 완료.

### **📅 3주차 (3/9 ~ 3/15): 프론트엔드 고도화 및 데이터 수집 엔진 구축 [IN PROGRESS]**
> **주간 목표: 완성된 UI에 실제 데이터를 수혈하고, 안정적인 데이터 수집 파이프라인을 구축한다.**

- **Day 1-4 (월-목): 프론트엔드 연동 및 UI 표준화 [COMPLETED]**
  - Pinia 기반 상태 관리 및 Axios 인터셉터(토큰 재발급) 연동.
  - 리스트/관심공고/캘린더/마이페이지 UI 구축 및 다크 모드 적용.
  - Tag 중심의 차세대 키워드 시스템 UI(자동완성 등) 구현.

- **Day 5-6 (금-토): 데이터 수집 엔진 개발 (Strategy A & B)**
  - **수집 파이프라인 구축:**
    - **LH (Open API):** `dsList` 배열의 `PAN_NM`, `CLSG_DT`, `DTL_URL`, `PAN_SS`, `AIS_TP_CD_NM`, `CNP_CD_NM` 등을 DTO로 매핑하여 수집.
    - **HUG (Open API):** 공고 만료 시 API 데이터가 내려오지 않는 특성을 고려하여, 기존 DB에 저장된 공고의 상태를 자체적으로 업데이트하는 로직 추가.
    - **SH (크롤러):** 서울주택도시공사 공고 리스트(`table.board-list`) 크롤링 및 상세 페이지 파싱.
    - **민간임대 (크롤러):** 서울시 청년안심주택 공고 리스트 크롤링 (상태 배지 및 마감일 추출).
  - **안정성 확보:** 크롤링 실패 시 Slack/Discord 알림 연동 및 에러 핸들링.
  - **데이터 파이프라인(B):** 공고 고유 번호(PAN_ID) 또는 URL 기반 중복 방지 로직 고도화.
  - **태그 자동 분류:** 수집된 비정형 데이터(지역, 유형)를 Tag 시스템에 맞게 자동 매핑.

- **Day 7 (일): 통합 테스트 및 UX 디테일링 (Strategy C)**
  - **Skeleton UI:** 데이터 로딩 시 체감 속도 향상을 위한 스켈레톤 화면 적용.
  - **Empty State:** 검색 결과나 관심 공고가 없을 때의 가이드 UI 보강.
  - 프론트-백엔드-크롤러 전체 흐름 통합 테스트.

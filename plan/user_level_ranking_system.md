# 🏆 사용자 통합 레벨링 및 명예의 전당 (Gamification System)

## 1. 개요 (Overview)
단순한 텍스트 배지 나열 방식에서 벗어나, 사용자의 모든 커뮤니티 활동을 **통합 경험치(EXP)**로 환산하고 이를 기반으로 **레벨(Level)**을 부여합니다. 레벨이 올라갈수록 닉네임의 시각적 효과가 화려해지며, 기간별 랭킹 시스템을 통해 신규 및 기존 유저 모두에게 강력한 활동 동기를 부여합니다.

---

## 2. 통합 레벨링 & 비주얼 진화 시스템 (Flamboyant Nickname)

### 2.1. 경험치(EXP) 획득 구조
모든 의미 있는 상호작용은 EXP로 변환됩니다.
*   **게시글 작성**: +10 EXP
*   **댓글 작성**: +3 EXP
*   **좋아요/도움돼요 받음**: +5 EXP (어뷰징 방지를 위해 일일 상한선 설정 필요)
*   **연속 출석**: +20 EXP (예: 7일 연속)
*   **특정 칭호(배지) 최초 획득**: +50 EXP (보너스)

### 2.2. 레벨 구간별 시각적 효과 (CSS Visual Tiers)
화면에 배지 아이콘을 여러 개 노출하는 대신, 닉네임 텍스트 자체의 CSS 이펙트를 진화시켜 깔끔함과 과시욕을 동시에 충족합니다.
*   **Tier 1 (Lv.1 ~ 10, 평민)**: 기본 텍스트
*   **Tier 2 (Lv.11 ~ 30, 시민)**: 은은한 **메탈릭(브론즈/실버) 그라데이션** 텍스트
*   **Tier 3 (Lv.31 ~ 50, 기사)**: **골드 그라데이션** + 입체적인 미세 텍스트 그림자
*   **Tier 4 (Lv.51 ~ 70, 귀족)**: 닉네임 주변에 은은하게 퍼지는 **빛나는 오라(Aura) 펄스 효과**
*   **Tier 5 (Lv.71 ~ 99, 왕족)**: 닉네임을 감싸는 **흐르는 빛 애니메이션 (Moving Gradient Border)**
*   **Tier 6 (Lv.100, 전설)**: 화려한 **파티클(Sparkle) 이펙트**와 특수 폰트 애니메이션

## 3. 칭호(Title) & 미니 프로필 UX

게시판 리스트나 상세 화면을 해치지 않기 위해, 상세한 칭호와 배지 정보는 사용자 상호작용(Hover/Touch) 시에만 노출합니다.

*   **대표 칭호 장착**: 사용자는 획득한 여러 배지 중 가장 마음에 드는 1개를 '대표 칭호'로 장착할 수 있습니다.
*   **상호작용 (PC: Hover / Mobile: Long Press)**:
    *   화려한 닉네임과 상호작용 시 **미니 프로필 툴팁** 노출.
    *   **구성 요소**: 장착 중인 대표 칭호, 현재 레벨(Lv.N), 다음 레벨까지의 EXP 게이지 바, 최근 획득한 배지 3개 요약.

## 4. 명예의 전당 (기간별 랭킹 시스템)

기존의 고레벨 유저들만 독식하는 '누적 랭킹'의 단점을 보완하여, 신규 유저도 단기간에 활약하면 돋보일 수 있는 **기간별 랭킹**을 도입합니다.

*   **기간별 랭킹 (일일 / 주간 / 월간 / 분기 / 연간)**:
    *   해당 기간 동안 **'가장 많은 EXP를 획득한 사용자'** 랭킹 보드.
    *   메인 페이지나 커뮤니티 사이드바에 '이번 주 활동왕' 형태로 노출.
*   **명예의 전당 (All-Time)**:
    *   역대 누적 EXP 랭킹 TOP 100.
*   **특별 보상**: 주간/월간 랭킹 1~3위 달성 시 한정판 특별 칭호("2월의 소통왕" 등) 부여.

## 5. 시스템 아키텍처 (Backend Design)

### 5.1. Entity 확장
*   `User`: `totalExp` (누적 경험치), `level` (현재 레벨), `equippedBadgeId` (장착 중인 대표 칭호) 컬럼 추가.
*   `ExpHistory`: 랭킹 집계를 위한 EXP 획득 이력 (누가, 언제, 어떤 이유로, 몇 점을 얻었는지 기록).

### 5.2. Service 로직
*   `ExpService`: 활동 발생 시 비동기 이벤트로 호출되어 `ExpHistory`를 기록하고, `User.totalExp`를 증가시키며 레벨업 여부를 판단.
*   `RankingService`: `ExpHistory` 테이블의 `createdAt`을 기준으로 일/주/월별 SUM(exp)를 계산하여 랭킹 캐싱(Redis 권장).

### 5.3. Frontend 컴포넌트화
*   전역 `<AnimatedNickname :user="author" />` 컴포넌트를 구축하여, 레벨에 따른 CSS Tier 클래스를 동적으로 할당하고 툴팁(미니 프로필) 이벤트를 일괄 관리.

# Week 3 - Day 5 (2026.02.27) 프론트엔드 및 시스템 고도화 상세 기록

## 1. 디자인 시스템 표준화 및 레이아웃 통합
- **PageHeader 공통 컴포넌트 도입**:
  - 모든 페이지(공고 리스트, 관심 공고, 캘린더, 마이페이지)의 헤더 규격을 `PageHeader.vue`로 통일.
  - 상단 아이콘, 제목, 통계 수치, Bio 설명글의 위치를 1px 오차 없이 일치시켜 시각적 안정감 확보.
- **전역 레이아웃 최적화**:
  - `scrollbar-gutter: stable`을 적용하여 세로 스크롤 생성 시 화면이 좌우로 흔들리는 현상(Layout Shift) 완벽 차단.
  - PC 버전 콘텐츠 가용 너비를 1200px로 고정하여 대화면에서도 정갈한 UI 유지.
- **다크 모드(Black Theme) 완성**:
  - CSS 변수를 기반으로 다크 모드 시 배경색, 카드색, 구분선색(`--divider-color: #555555`) 등을 정밀하게 조정.
  - 정렬(Select) 박스 등 폼 요소가 다크 모드에서 가독성이 떨어지는 문제를 테마별 스타일 강제 적용으로 해결.

## 2. 공고 리스트(Notice List) 기기별 이원화 및 UI 개편
- **기기별 최적화 레이아웃 (Device-Specific UI)**:
  - **PC**: 전문적인 **테이블 리스트** 형식 적용. [기관 | 지역 | 공고명 | 상태 | 디데이 | 접수일자 | 공고태그] 순서의 7개 컬럼 구성.
  - **Mobile**: **슬림 카드** 형식 적용. 내용을 숨기고 제목과 태그, 날짜 배지 위주로 얇게 설계하여 한 화면에 더 많은 정보를 노출.
- **데이터 바인딩 및 정렬 로직**:
  - `상태(예정/접수중/마감)` 및 `디데이(D-Day/D+/D-)` 실시간 계산 로직 구현.
  - 제목에서 지역과 공급 유형을 파싱하여 별도 컬럼 및 해시태그로 자동 노출하는 기능 강화.
- **사용자 편의성(UX) 강화**:
  - **PC 페이징 도입**: 페이지당 15개 항목씩 노출하며 하단 번호를 통해 명확한 위치 이동 가능.
  - **스마트 스크롤(Anchor)**: 페이지 전환 시 화면이 자동으로 검색창 최상단 위치로 부드럽게 튕겨 올라가도록 설정.
  - **반응형 감지**: `matchMedia` API를 도입하여 브라우저 창 조절 및 개발자 도구의 디바이스 토글 시에도 즉각적으로 레이아웃이 전환되도록 개선.

## 3. 관심 공고(My Interests) 기능 대폭 고도화
- **그리드 대시보드 디자인**:
  - 공고 이미지를 대신하여 기관별 고유 그라데이션 배경과 큼직한 D-Day 타이포그래피를 적용하여 가시성 극대화.
  - 마감된 공고는 흑백 톤 처리 및 D-Day 빨간색 강조로 시각적 경고 효과 부여.
- **참고 URL 관리 시스템**:
  - 공고별로 블로그, 매물 사이트 등 여러 링크를 저장할 수 있는 기능 추가.
  - 저장된 링크는 클릭 시 즉시 이동 가능한 **배지(Badge)** 형태로 표시(긴 주소는 말줄임 처리).
  - 새로운 링크는 하단 입력창을 통해 즉시 추가하고 통합 저장하는 스마트 인터페이스 구축.
- **모바일 특화 인터랙션**:
  - **Touch Hold (꾹 누르기)**: 누르고 있는 동안에만 메모와 마감일 정보를 보여주는 투명 오버레이 노출.
  - **Heart Toggle**: 그리드 우측 상단 하트 버튼으로 상세 창을 열지 않고도 빠른 스크랩 취소 지원.

## 4. 태그(Tag) 중심의 차세대 키워드 시스템 구축
- **아키텍처 혁신**: 
  - 자유 텍스트 방식의 구형 Keyword 시스템을 폐기하고, 통제된 사전 방식의 **Tag 시스템** 도입.
  - `Tag`(마스터), `NoticeTag`(공고 연결), `UserTag`(사용자 선호)의 3단 데이터 구조 설계.
- **마이페이지 알림 키워드 UI 개편**:
  - **자동완성 검색**: 태그 테이블 내의 데이터를 실시간 검색하여 선택하는 방식 적용 (REGION, TYPE 등 타입 뱃지 표시).
  - **키보드 내비게이션**: 검색 결과에서 방향키(↑, ↓)로 탐색하고 엔터(Enter)로 즉시 등록하는 편의 기능 구현.
  - **칩(Chip) 관리**: 등록된 태그를 세련된 해시태그 칩 형태로 나열하고 즉시 삭제 가능하도록 구현.

## 5. 데이터 및 인프라 기반 강화
- **샘플 데이터 고도화**:
  - 50개의 실제와 유사한 비정형 공고 데이터와 정교한 태그 매핑 정보가 포함된 `init_data.sql` 구축.
  - MySQL 외래 키 제약 조건을 고려한 안전한 데이터 초기화 및 삽입 스크립트 작성.
- **외부 서버 연동 테스트**:
  - Cloudflare Tunnel을 활용하여 WSL2 환경의 로컬 서버를 외부로 노출하는 인프라 테스트 진행.
  - 외부 접속 시 발생할 수 있는 CORS 정책 및 Axios BaseURL 설정 점검 완료.

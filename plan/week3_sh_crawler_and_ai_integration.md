# Week 3: SH 크롤러 구현 및 Gemini AI 연동 (슈퍼 배치 고도화)

## 📅 작업 기간: 2026-03-03

## 🎯 주요 목표
SH 공고 데이터의 비정형성을 극복하고, Gemini 2.5 Flash-Lite 모델의 무료 티어 제한(일일 20회 호출) 내에서 1,580개 전체 공고의 마감일을 완벽하게 수집합니다.

---

## ✅ 완료된 작업

### 1. 슈퍼 배치(Super Batching) 로직 구현
*   **할당량 극복 (Quota Optimization)**: Gemini 2.5 Flash-Lite의 일일 호출 제한(20회)을 준수하기 위해, 공고 **80개를 하나의 묶음**으로 처리하도록 개조.
*   **효율성**: 전체 1,580개 공고를 단 20번의 AI 호출만으로 전수 분석 가능하도록 설계.
*   **데이터 축적(Buffering)**: 여러 페이지를 순회하며 공고를 버퍼에 모았다가 임계값(80개) 도달 시 일괄 분석 수행.

### 2. Gemini 2.5 Flash-Lite AI 연동 최적화
*   **최신 모델 적용**: 성능과 속도가 향상된 `gemini-2.5-flash-lite` 모델 사용.
*   **지능형 프롬프트 v2**: 
    *   비정형 텍스트 내의 다양한 날짜 형식(2024.3.1, 24년 3월 등)을 인식하여 `yyyy-MM-dd`로 변환.
    *   공고 제목과 본문을 함께 전달하여 문맥 파악 능력 극대화.
*   **JSON 응답 안정화**: 마크다운 태그 제거 및 JSON 파싱 보정 로직 추가.

### 3. 시스템 안정성 및 비용 절감 로직
*   **중복 호출 방지**: AI 분석 전 DB를 확인하여, 이미 마감일(`deadline`)이 존재하는 공고는 분석 대상에서 제외 (API 할당량 보존).
*   **스마트 중단(Circuit Breaker)**: 429(할당량 초과), 503(서버 과부하), 404(경로 오류) 발생 시 크롤링 루프를 즉시 중단하여 리소스 낭비 방지.
*   **타임아웃 강화**: SH 서버의 지연 응답에 대비하여 상세 페이지 수집 타임아웃을 **10초**로 연장.

### 4. 데이터 모델 및 스케줄링
*   **announcementDate 도입**: 공고 게시일 필드를 추가하여 LH와 데이터 규격 통일.
*   **자동 동기화**: 매일 새벽 2시 5분 스케줄러를 통해 신규 공고 발생 시에만 AI가 작동하도록 최적화.

---

## 📊 현재 진행 상황 및 성과
*   **수집 효율**: 기존 방식 대비 API 호출 횟수를 1/8로 절감하면서도 동일한 데이터 수집 가능.
*   **운영 안정성**: 일일 한도 초과 시에도 데이터 유실 없이 다음 날(리셋 후) 자동으로 나머지 작업을 수행할 수 있는 "이어하기" 구조 완성.
*   **데이터 정밀도**: 규칙 기반 수집의 한계를 AI의 문맥 분석으로 해결하여 마감일 오차 최소화.

---

## 🚀 향후 계획
1.  **AI 분석 실패 케이스 분석**: `NONE`으로 리턴되는 이미지 위주 공고들에 대한 별도 수집 전략(OCR 등) 검토.
2.  **데이터 품질 모니터링**: AI가 추출한 마감일이 실제 공고와 일치하는지 정기적인 샘플링 검사 실시.
3.  **수집 알림 고도화**: 크롤링 중단 사유(할당량 초과 등)를 관리자에게 즉시 알리는 알림 기능 추가.
